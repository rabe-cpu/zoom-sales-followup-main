# DOCX Output Rules

## Word出力

- `.docx` で出す。
- 元ファイルは変更しない。
- 新しい出力フォルダまたは新しいファイル名にする。
- 複数商談の場合は、個別docxとまとめdocxのどちらが自然か判断する。
- 顧客送付用docxと社内確認用docxは分ける。
- 顧客送付用docxに営業口調抽出、評価ログ、残リスク、社内確認情報を混ぜない。
- `[黄色:...]` や `[/黄色]` は本文に出さず、Wordのハイライトへ変換する。

## 黄色欄

黄色欄は営業が最後に触る箇所だけ。

原則:

1. 冒頭の確認差し替え欄
2. `ZOOM URL：〇〇〇`

必要がある場合だけ追加:

- 次回日時が未確定
- 営業担当署名が未確定
- 動画URLが未確定

## 推奨Word構成

1. 営業後送付メール
2. 営業チェック欄
3. ソーシャルスタイル別全文メール案（社内確認用）
4. 商談フィードバック要素
5. 最終確認

顧客送付用は1のみを基本にする。2以降は社内確認用に分ける。
ソーシャルスタイル別メール案は、顧客送付用docxには入れない。
営業口調抽出、季語調査結果、参考動画選定理由は生成時の内部根拠として使い、docxの独立セクションには出さない。
評価ログと残リスクは、顧客送付用docxにも社内確認用docxにも出さない。

## ソーシャルスタイル別メール案

社内確認用docxには、`knowledge/12_social_style_email_variants.md` に従い、以下の4スタイルを同じ構成で入れる。

- Driver / Driving
- Analytical
- Amiable
- Expressive

各スタイルには、件名から署名・固定資料URL・固定フォームURLまで含む全文メール案、顧客反応シグナル、営業担当が選ぶ目安、次回商談での質問例、そのまま使える返答例、価格・費用質問への返し方、避ける言い方、伝え方メモ、次の一手、ベンチマーク営業トーク、文脈接続メモ、リスク注意を入れる。「この文面が効く理由」は見出し・本文とも独立項目として出さない。
差し替え段落だけの出力は不合格。
これは顧客タイプの判定ではなく、営業担当が送信前に表現を選ぶための社内参考資料として扱う。
全文メール案は読みやすさを優先する。1段落は原則1〜2文、長くても120〜160字程度にし、商談背景、判断軸、作業内容、参考動画、次アクションを1段落に混ぜない。判断軸、作業内容、確認事項は2〜4項目までの短い箇条書きを使ってよい。

## 商談フィードバック要素

社内確認用docxの「商談フィードバック要素」は、`sales-analysis-app-openai-next` の分析出力に近い厚みで出す。
ただし、docx上では内部キー名を見せない。`overallSummary`、`hiddenNeeds`、`name=`、`item=`、`theme=`、`benchmarkCoach.script` のような表示は不合格。
トップ営業マンが目の前で営業担当に指導している形で、「この商談はこう見る」「次回はこう聞く」「この場面ではこう返す」という自然文を中心にする。箇条書きは次回質問、確認項目、NG例などの実務チェックに限る。

必須ブロック:

1. 総合概要
2. 顧客インサイト
3. 認知バイアス
4. 期待値のズレ
5. 良かった点
6. 改善ポイント
7. AIコーチングカード
8. 再現する勝ち筋
9. 商談フェーズ
10. 顧客シグナル
11. 次の一手
12. ベンチマーク営業再現
13. 文脈接続
14. 属性別対応

AIコーチングカードは4枚を基本にし、各カードに `theme / scene / insight / issue / strategy / script / outcome` を入れる。
再現する勝ち筋には、勝ち筋名、使う場面、顧客シグナル、トップ営業の動き、コピートーク、なぜ効くか、NG例、次回実践方法を入れる。
`knowledge/benchmark_playbooks/suzue_benchmark.md` を必ず参照し、文字起こしベースの営業型として `benchmarkCoach`、`winningPatterns`、`phasePlaybooks`、`customerAttributePlaybooks` に反映する。音声・映像コーチング、声色・話速評価、録音練習、模写練習メニューは出力しない。
評価ログ、残リスク、スコアは入れない。

表示例:

```text
### 認知バイアス

トップ営業の指導:
このお客様は、初期費用そのものよりも「今の与信状態で動いて失敗しないか」を強く見ています。ここで「大丈夫です」とまとめると逆に不信感が出ます。次回は、月額、必要自己資金、審査可否を分けて、「どこが一番引っかかっていますか」と先に聞いてください。

そのまま使える台詞:
「金額だけで見ると判断が難しくなるので、月々の支払い、黒字化までに必要な資金、審査の扱いを分けて整理しましょう。今いちばん気になるのはどこですか。」
```

## スクリプト

`scripts/create_followup_docx.py` は、構造化JSONからWordを作る。

最小JSON:

```json
{
  "title": "営業後送付メール",
  "subject": "件名",
  "recipient": "宛名",
  "body": ["本文1", "本文2"],
  "yellow_fields": ["ZOOM URL：〇〇〇"],
  "social_style_variants": {
    "Driver / Driving": "件名から署名まで含む全文メール案",
    "Analytical": "件名から署名まで含む全文メール案",
    "Amiable": "件名から署名まで含む全文メール案",
    "Expressive": "件名から署名まで含む全文メール案"
  },
  "deal_feedback": {"次の一手": "送信後の確認観点"},
  "final_check": {"顧客送付用本文": "確認済み", "黄色箇所": "確認済み"}
}
```
