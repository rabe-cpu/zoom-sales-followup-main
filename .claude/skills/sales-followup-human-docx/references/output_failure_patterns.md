# Output Failure Patterns

## 目的

営業後送付メールが、AIの作業ログや資料メモのように見える失敗を防ぐ。

## 失敗パターンと修正

### 1. タグが本文に出る

NG:

- `[黄色:営業確認]`
- `[/黄色]`
- `[黄色:参考動画URL]〇〇〇[/黄色]`

対応:

- タグ文字は本文に出さない。
- 黄色はWordの装飾で表現する。
- docx生成時は `scripts/create_followup_docx.py` の黄色タグ除去/ハイライト変換を使う。

### 2. `【】` 見出しが多い

NG:

- `【参考動画】`
- `【次回個別面談】`
- `【プラン・シミュレーションについて】`

対応:

- 顧客向け本文では、見出しを文章に溶かす。
- 例: `参考になりそうな動画も1本お送りします。`
- 見出しを使う場合は1から2個まで。

### 3. 箇条書きが多い

NG:

- 5個以上の長い箇条書き
- 事実整理をそのまま貼る

対応:

- 本文では2から3項目まで。
- それ以上は「資料にまとめています」に寄せる。
- 重要な判断軸だけ文章で触れる。

### 4. 動画が出ない

NG:

- `参考動画URL：〇〇〇`
- URLなしで「動画をお送りします」と書く

対応:

- 動画カタログから、動画タイトル、URL、選定理由、見る観点を必ず出す。
- 動画URLがない場合は完成扱いにしない。
- `〇〇〇` を許すのは原則 `ZOOM URL：〇〇〇` のみ。

### 5. 顧客送付用と社内確認用が混ざる

NG:

- 顧客送付用Wordに評価ログ、営業口調抽出、残リスクが続けて入る

対応:

- 顧客送付用docx: メール本文だけ
- 社内確認用docx: 黄色欄、評価ログ、口調抽出、残リスク
- まとめ版が必要な場合も、ページ区切りと見出しで明確に分ける。

### 6. 日付が破綻する

NG:

- 送付日より前の日付を「次回」として本文に入れる

対応:

- 送付日と次回予定の前後関係を確認する。
- 過去日付ならblocking。
- 不明な場合は営業確認欄に回し、本文では断定しない。

### 7. 金額が漢数字になる

NG:

- `月々七万円ほど`
- `総額三百三十万円ほど`
- `二万二千円ほどから五千五百円ほど`

対応:

- 金額は必ずアラビア数字に直す。
- `月々7万円ほど`
- `総額330万円ほど`
- `22,000円ほどから5,500円ほど`

## 最終チェック

顧客送付用本文に以下が残っていたら未完成。

- `[黄色`
- `[/黄色]`
- `参考動画URL：〇〇〇`
- 金額の漢数字表記（例: `七万円`、`三百三十万円`、`二万二千円`）
- 5個以上の連続箇条書き
- 3個以上の `【】` 見出し
- 評価ログ、残リスク、営業口調抽出
