# Output Failure Patterns

## 目的

営業後送付メールが、AIの作業ログや資料メモのように見える失敗を防ぐ。

## 失敗パターンと修正

### 1. タグが本文に出る

NG:

- `[黄色:営業確認]`
- `[/黄色]`
- `[黄色:参考動画URL]〇〇〇[/黄色]`

対応:

- タグ文字は本文に出さない。
- 黄色はWordの装飾で表現する。
- docx生成時は `scripts/create_followup_docx.py` の黄色タグ除去/ハイライト変換を使う。

### 2. `【】` 見出しが多い

NG:

- `【参考動画】`
- `【次回個別面談】`
- `【プラン・シミュレーションについて】`

対応:

- 顧客向け本文では、見出しを文章に溶かす。
- 例: `参考になりそうな動画も1本お送りします。`
- 見出しを使う場合は1から2個まで。

### 3. 箇条書きが多い

NG:

- 5個以上の長い箇条書き
- 事実整理をそのまま貼る

対応:

- 本文では2から3項目まで。
- それ以上は「資料にまとめています」に寄せる。
- 重要な判断軸だけ文章で触れる。

### 4. 動画が出ない

NG:

- `参考動画URL：〇〇〇`
- URLなしで「動画をお送りします」と書く

対応:

- knowledge/video_catalog.md から、動画タイトル、実際のYouTube URL、選定理由、見る観点を必ず出す。
- 動画URLがない場合は完成扱いにしない。
- `〇〇〇` を許すのは原則 `ZOOM URL：〇〇〇` のみ。

### 5. 顧客送付用と社内確認用が混ざる

NG:

- 顧客送付用Wordに社内確認情報や残リスクが続けて入る

対応:

- 顧客送付用docx: メール本文だけ
- 社内確認用docx: 黄色欄、ソーシャルスタイル別全文メール案、商談フィードバック要素、最終確認、残リスク
- まとめ版が必要な場合も、ページ区切りと見出しで明確に分ける。

### 5a. ソーシャルスタイル別案が一部だけになる

NG:

- 冒頭の差し替え段落だけ
- 判断軸整理だけ
- 「Driver向けは短めに」などの説明だけ
- 4スタイルのうち1〜2個だけ

対応:

- Driver / Driving、Analytical、Amiable、Expressive の4つを必ず出す。
- 各スタイルは、件名、宛名、本文、参考動画、ネクストアクション、署名、固定資料URL、固定フォームURLまで含む全文メールにする。
- 各スタイルの後ろに、商談フィードバック要素（顧客反応シグナル、次回質問、返答例、価格質問対応、避ける言い方、伝え方メモ、次の一手、ベンチマーク営業トーク、文脈接続メモ）を営業メモとして付ける。

### 6. 資料・フォームURLが統一されていない

NG:

- `【個別相談会資料はこちらです】`
- `【検討結果入力はこちらからご入力お待ちしております】`
- 資料URLやフォームURLが別URL、または `〇〇〇` のまま

対応:

- 顧客送付用では以下に統一する。

```text
【個別相談会資料】
https://nextjp.co.jp/document/access/kobetsu_japan6.pdf

【検討結果入力フォーム】
https://docs.google.com/forms/d/e/1FAIpQLSd7K5Ki8TOdRVeE_zEZroSiDfqTFyS5RKf6HRXAa4_WWxg7iA/viewform
```

この2つは営業差し替え箇所ではないため、黄色にしない。

### 7. 日付が破綻する

NG:

- 送付日より前の日付を「次回」として本文に入れる

対応:

- 送付日と次回予定の前後関係を確認する。
- 過去日付ならblocking。
- 不明な場合は営業確認欄に回し、本文では断定しない。

### 8. 金額が漢数字になる

NG:

- `月々七万円ほど`
- `総額三百三十万円ほど`
- `二万二千円ほどから五千五百円ほど`

対応:

- 金額は必ずアラビア数字に直す。
- `月々7万円ほど`
- `総額330万円ほど`
- `22,000円ほどから5,500円ほど`

## 最終チェック

顧客送付用本文に以下が残っていたら未完成。

- `[黄色`
- `[/黄色]`
- `参考動画URL：〇〇〇`
- `【個別相談会資料はこちらです】`
- `【検討結果入力はこちらからご入力お待ちしております】`
- 金額の漢数字表記（例: `七万円`、`三百三十万円`、`二万二千円`）
- 5個以上の連続箇条書き
- 3個以上の `【】` 見出し
- 社内確認情報、残リスク、営業口調抽出
