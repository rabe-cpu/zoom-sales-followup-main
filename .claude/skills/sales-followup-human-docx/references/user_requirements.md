# User Requirements

## 絶対に反映すること

- 元ファイルは変更せず、新しいフォルダまたは新しいWordファイルを作る。
- 出力はWord/docxを基本にする。
- 営業が最後に入力・確認する箇所だけ黄色にする。
- 黄色欄は増やしすぎない。原則は冒頭の確認差し替え欄と `ZOOM URL：〇〇〇`。
- `ZOOM URL：〇〇〇` は明るい色または黄色で明示する。
- 顧客との面談で実際に話した内容を冒頭に入れる。
- 冒頭は、ただ短く触れるだけでなく、2から4文で少し話を広げる。
- 営業担当のMTG中の口調を毎回抽出し、それを反映する。
- 口調は顧客発話ではなく、営業担当の実発話だけを根拠にする。
- `かと`、`かなと`、`できればと`、`見ていただけると` は営業担当が言っている場合だけ使う。
- 発話にある場合でも、余白表現は1通あたり0から1回程度にする。
- `思います` が連続しないように、文末を分散する。
- `長岡さまには合っているのではないかと`、`時間があるときに見ていただけると。` のような柔らかさは、営業担当の口調に合う場合だけ使う。
- 季語・時候の挨拶は毎回調査する。
- 送付日、月内区分、二十四節気、参照元を残す。
- 参考動画はURLだけを貼らず、URL前に選定理由、URL後に見る観点を書く。
- 金額は必ずアラビア数字で書く。`七万円`、`三百三十万円`、`二万二千円` のような金額の漢数字表記は使わない。
- ペルソナを構築し、人間っぽく受け止められるかを評価する。
- 評価エージェントを使い、90点以上になるまで改善する。
- 最後に全体チェックし、足りない点があればAIで改善する。

## 表現ルール

- `伝わってまいりました` は一律禁止ではない。ただし、営業担当の実発話、雛形、または営業口調に合う根拠がある場合だけ使う。
- 根拠が薄い場合は `伝わってきました`、`感じました`、`受け取りました` など自然な表現に寄せる。
- 顧客が言っていない感情や成果期待を断定しない。
- 成果保証、収益保証、過度な期待を避ける。
- 顧客向けの最終文では、意図的な誤字は入れない。人間味は会話反映と営業口調で出す。

## 人間味の作り方

- 会話で出た固有の話題を冒頭に入れる。
- 営業担当が実際に使った言い回しだけを拾う。
- 文末を `思います` だけにしない。
- 1文を長くしすぎず、自然に文を分ける。
- 丁寧すぎるAI語を避ける。
- メール全体をきれいに整えすぎず、相手に合わせて少しだけ余白を残す。
