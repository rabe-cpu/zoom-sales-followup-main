# 04 Email Generation Rules

## 基本構成

1. 件名
2. 宛名
3. 挨拶・名乗り
4. お礼
5. 商談で話した内容の回収
6. 判断軸の整理
7. 資料・プラン・シミュレーション案内
8. 参考動画
9. ネクストアクション
10. 資料送付
11. 安心導線
12. 季節の結び
13. 締め

## 冒頭の会話反映

冒頭は、次の3つを入れる。

- 顧客が話した背景
- 顧客が気にしていた判断軸
- 営業が受け止めた自然な一文

例:

`本日お話を伺う中で、上田さまは新しいことをただ増やしたいというより、飲食店やキッチンカーで今ある人員・空き時間をうまく使えるかを、かなり現実的に見られているんだなと感じました。`

## 参考動画の書き方

自然なのは、URLの前後に短く説明を入れる形。

構成:

1. URL前: なぜこの動画を選んだか
2. URL
3. URL後: どこを見ると判断材料になるか

URL後の説明を省くと、営業がちゃんと選んだ感じが弱くなる。

ただし、URL後を長く書くと宣伝っぽくなるため1文で止める。

## ソーシャルスタイル別メール案（社内確認用）

社内確認用ドキュメントには、`knowledge/12_social_style_email_variants.md` に従い、4つのソーシャルスタイル別メール案を追加する。

重要:

- 顧客の会話からソーシャルスタイルを判定しない。
- `この顧客はDriver`、`Analytical型と推定` のように断定しない。
- 同じ商談事実を、Driver / Driving、Analytical、Amiable、Expressive の4つの伝え方に変換する。
- 顧客送付用メールには、4スタイル別案や社内フィードバックを混ぜない。
- 4スタイル別案は、営業担当が送信前に表現を選ぶための社内確認用として残す。
- 各スタイルには、メール案、この文面が効く理由、営業担当が選ぶ目安、次回商談での質問例、価格・費用質問への返し方、避ける言い方、伝え方メモを含める。

## 資料・検討結果入力フォームの固定表記

顧客送付用メールで案内する資料URLと検討結果入力フォームは、以下に統一する。
別の見出し、別URL、`〇〇〇` の仮置き、黄色マーカーは使わない。

```text
【個別相談会資料】
https://nextjp.co.jp/document/access/kobetsu_japan6.pdf

【検討結果入力フォーム】
https://docs.google.com/forms/d/e/1FAIpQLSd7K5Ki8TOdRVeE_zEZroSiDfqTFyS5RKf6HRXAa4_WWxg7iA/viewform
```

この2つは営業が差し替える箇所ではないため、黄色にしない。
`【】` 見出し数のチェックでは、この2つの定型ラベルは許容する。

## ネクストアクション

依頼文は途中で止めない。

NG:

- `状況だけ一度伺えればと。`
- `資料を見ていただけると。`
- `進められればと。`

OK:

- `状況だけ一度伺えますと助かります。`
- `お時間あるときにご覧いただけますと助かります。`
- `一つずつ確認させてください。`
- `こちらのメールにご返信いただいてもよろしいでしょうか。`

## 黄色箇所

### 記法（最重要・必ず守る）

黄色マーカーは**必ず `[黄色]` と `[/黄色]` で囲む**。後処理（Google Docs変換時）にこのタグの中身を実際の黄色ハイライトに変換するため、この形式が必須。
**`🟡` 絵文字・`【】`・「説明表だけで示す」などの独自記法は使わない**（変換されず黄色にならない）。

黄色にするもの:

- 冒頭の会話反映で、営業本人の温度感が必要な箇所 → `[黄色]…[/黄色]`
- **`ZOOM URL：〇〇〇` … 本文に出てくる全箇所（複数あれば全部）を必ず `[黄色]ZOOM URL：〇〇〇[/黄色]` で囲む**
  - 次回相談会URL・個別面談URL・次回ZOOM URL等、営業が送付前に差し替える必要があるZOOM系URLは1つ残らず黄色にする
  - （次回ZOOM予定が商談内に無い場合のみ、ZOOM URL自体を本文に入れない＝黄色も無し）
- 事実確認が必要な固有情報

黄色にしないもの:

- AIが商談から確実に抽出できる事実
- 動画URL
- 一般的な季語
- 定型の締め

## 禁止・注意表現

- `伝わってまいりました`
- `必ず成果が出ます`
- `安心して稼げます`
- `誰でも簡単に`
- 顧客が言っていない不安・感情の断定
- 営業担当の発話にない余白表現

## 数字・金額表記

金額は必ずアラビア数字で書く。漢数字で金額を書かない。

OK:

- `月々7万円ほど`
- `総額330万円ほど`
- `自己資金30万円ほど`
- `半年間のランニング費用が月々22,000円ほどから5,500円ほどになる`

NG:

- `月々七万円ほど`
- `総額三百三十万円ほど`
- `自己資金三十万円ほど`
- `二万二千円ほどから五千五百円ほど`

日数・時間・回数・割合も、原則としてアラビア数字で統一する。
ただし、自然な言い回しの `一度`、`一緒に`、`一つずつ` など金額・数量として扱わない語は漢字のままでよい。

## メール本文のフォーマット（最重要）

営業後送付メールは「**段落形式の本文**」で書く。報告書フォーマット・社内文書フォーマットにしない。

### 禁止フォーマット

- **箇条書きを本文に使わない**（`・` `-` `1.` `①` などの行頭記号、複数行のリスト構造）
  - 例外：参考動画URL／資料URL／署名直下のURLリンクなど「URLそのもの1行」だけはOK
- **見出し・Heading スタイルを使わない**（`#` `##` `■` `【見出し】` を本文構造として使わない）
  - 例外：件名・`【個別相談会資料】`、`【検討結果入力フォーム】`のような定型ラベル・黄色マーカー対象の `[黄色`〜`[/黄色]`はOK
- **宛名は冒頭1回のみ**。Title欄／Heading 1／本文中で繰り返さない
- **ドキュメントタイトル化しない**（「営業後送付フォロー」「個別面談フォローアップ」のような社内タイトルは付けない）

### 正しいフォーマット

- 件名 → 宛名 → 挨拶・名乗り → お礼 → 会話反映（2〜4文の段落） → 判断軸の整理（段落） → 資料・シミュ案内（段落） → 参考動画（短文＋URL＋短文） → ネクストアクション（段落） → 安心導線（1〜2文） → 季語の結び → 締め → 署名 → 固定の資料URL・検討結果入力フォーム
- 各ブロックは **平文の段落** で繋ぐ。読点・接続詞で温度感を作る。
- 箇条書きにしたくなる「3つの要点」は、`〇〇のところ、〇〇のところ、そして〇〇のところ` のように文章でつなぐ。

### 悪い例 → 良い例

NG（報告書化）:

```
上記を踏まえ、判断軸として大きい順に下記のあたりかと思います。
・1日1〜2時間ほどの稼働で、月15〜20万円規模の収益を目指している前提
・昨年ほどで支払いと売上が立っている見通し
・在庫を持たず、注文発生時に対応する仕組みであること
```

OK（段落化）:

```
本日のお話の中で、判断軸として大きいのは、1日1〜2時間ほどの稼働で月15〜20万円規模を狙えるかどうか、半年ほどで支払いと売上のバランスが取れる見通しが立つかどうか、そして在庫を持たずに注文ベースで動かせる仕組みかどうか、というあたりだと受け取りました。
```

NG（タイトル＋Heading化）:

```
営業後送付フォロー
井上由実子さま

# 営業後送付フォロー
井上由実子さま

お世話になっております。
```

OK（メール文として）:

```
井上由実子さま

お世話になっております。
株式会社NEXT（総合物販システム＿アクセス）の笠井でございます。
```

NG（宛名重複）:

```
井上由実子さま
〔件名行〕
井上由実子さま
お世話になっております。
```

OK（宛名1回）:

```
【件名】本日はありがとうございました／契約お手続きについて

井上由実子さま

お世話になっております。
```
