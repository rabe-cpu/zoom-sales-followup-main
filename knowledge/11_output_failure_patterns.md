# 11 Output Failure Patterns

## 目的

営業後送付メールが、AIの作業ログや資料メモのように見える失敗を防ぐ。

## 必ず防ぐ失敗

- `[黄色:営業確認]` や `[/黄色]` が本文に出る
- `【参考動画】`、`【次回個別面談】` などの見出しが多すぎる
- 面談整理が5個以上の長い箇条書きで並ぶ
- `参考動画URL：〇〇〇` のまま完成扱いになる
- 顧客送付用Wordに評価ログ、営業口調抽出、残リスクが混ざる
- 送付日より前の日付を「次回」として本文に入れる

## 修正方針

1. 顧客送付用docxと社内確認用docxを分ける。
2. 黄色はWord装飾で表現し、タグ文字を本文に出さない。
3. 参考動画は実URL、タイトル、選定理由、見る観点を必須にする。
4. 箇条書きは2から3個まで。多い場合は文章化する。
5. `【】` 見出しは必要最小限にし、できるだけ文章に溶かす。
6. 日付の前後関係を必ず確認し、過去日付ならblockingにする。

## Ops-Formatting Agentの追加確認

顧客送付用本文に以下が残っていたらblocking。

- `[黄色`
- `[/黄色]`
- `参考動画URL：〇〇〇`
- 5個以上の連続箇条書き
- 3個以上の `【】` 見出し
- `評価ログ`
- `残リスク`
- `抽出した営業口調`
